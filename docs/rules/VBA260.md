# VBA260 — Member not found in type

**Severity:** 🔴 error    **Category:** `member_access`    **Phase:** 2.6

## Description

After resolving a dotted member chain `a.b.c[(i)].d` the analyser could not find the named member on the inferred type at that hop. The chain walker preserves element types through array indices, function-call returns, and property-get returns of arbitrary depth, so this is a genuine typo or missing declaration.

## Failing example

```vb
Private Type Cell: val As Long: End Type
Private Type Row: cells() As Cell: End Type
Sub S()
    Dim r As Row
    Debug.Print r.cells(0).bogus    ' Cell has no `bogus`
End Sub
```

## Compliant example

```vb
Private Type Cell: val As Long: End Type
Private Type Row: cells() As Cell: End Type
Sub S()
    Dim r As Row
    Debug.Print r.cells(0).val
End Sub
```

## How to fix

Fix the spelling or add the member to the UDT / class. If the parent type is an `Object` / `Variant` bag the analyser deliberately skips this check — give the variable a concrete declared type to opt back in.
